# ADR-002: 一期采用三方协同 + 供应商门户轻量版

## Status

Accepted

## Date

2026-06-07

## Context

系统服务对象已明确为：

- 招标人 / 采购人内部
- 招标代理机构
- 投标供应商门户

但如果一期直接建设完整公共电子交易平台，会立刻引入：

- CA 签章
- 文件加解密
- 在线解密开标
- 隔离式电子评标
- 监管平台对接

这会显著放大范围和交付风险。

## Decision

一期采用三方协同方案，但将供应商门户控制为轻量版，只覆盖：

- 注册 / 认证
- 项目查看
- 报名
- 投标文件提交
- 澄清响应
- 结果查看

## Alternatives Considered

### 一期只做内部系统

- Pros
  - 范围最小
- Cons
  - 无法满足供应商在线协同的核心诉求

### 一期直接做完整公共交易平台

- Pros
  - 目标最完整
- Cons
  - 合规、技术、安全复杂度过高
  - 与当前仓库基础能力不匹配

## Consequences

- 一期可实现真实三方协同
- 复杂电子交易能力后置到增强版
- 门户设计要预留后续升级空间，但当前不为未确认需求过度设计
