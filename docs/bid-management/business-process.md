# 招投标管理系统业务流程设计

## 参与方

- 招标人 / 采购人
  - 发起项目、审核关键决策、确认定标结果
- 招标代理机构
  - 代办流程、维护文件、发布公告、组织开评标
- 投标供应商
  - 报名、提交投标文件、响应澄清、查看结果
- 评标专家
  - 一期默认作为内部受控角色参与评分，不单独建设复杂专家库闭环

## 主流程总览

```mermaid
flowchart LR
  A["项目建档"] --> B["标段规划"]
  B --> C["招标文件与公告发布"]
  C --> D["供应商报名/受邀"]
  D --> E["投标文件提交"]
  E --> F["开标记录"]
  F --> G["评标打分与汇总"]
  G --> H["定标确认"]
  H --> I["结果归档与通知"]
```

## 分阶段说明

### 1. 项目建档

- 输入
  - 项目名称、采购方式、预算、组织归属、经办团队
- 输出
  - 项目编号
  - 项目责任人
  - 初始状态 `DRAFT`
- 关键规则
  - 项目未发布前允许编辑基础信息
  - 项目至少包含一个标段才能进入下一阶段

### 2. 标段规划

- 输入
  - 标段名称、范围、预算、报名时间、开标时间
- 输出
  - 标段清单
  - 每个标段的投标与评标边界
- 关键规则
  - 标段是投标、开标、评标、定标的最小执行单元
  - 标段信息冻结后才允许发布招标文件

### 3. 招标文件与公告发布

- 输入
  - 招标方案
  - 招标文件
  - 公告正文与附件
- 输出
  - 公告发布记录
  - 招标文件版本
  - 供应商可见的项目条目
- 关键规则
  - 发布后形成版本快照
  - 澄清文件必须建立与原版本的关联

### 4. 供应商报名 / 受邀

- 输入
  - 报名申请或邀请记录
  - 供应商基础资料
- 输出
  - 报名状态
  - 资格初审结果
- 关键规则
  - 一期允许人工审核资格，不先做自动准入规则引擎
  - 报名通过后才允许提交投标文件

### 5. 投标文件提交

- 输入
  - 供应商投标文件
  - 投标报价、联系人、附件清单
- 输出
  - 投标提交记录
  - 回执编号
  - 截止时间前的版本记录
- 关键规则
  - 支持截止前多次提交，以最后一次有效版本为准
  - 超过截止时间禁止新提交

### 6. 开标

- 输入
  - 已截止的投标记录
  - 现场开标信息
- 输出
  - 开标记录
  - 唱标结果
  - 异常登记
- 关键规则
  - 开标后投标文件不再允许修改
  - 异常必须留痕并具备责任人

### 7. 评标

- 输入
  - 评标模板
  - 专家评分
  - 澄清意见
- 输出
  - 评分明细
  - 汇总结果
  - 废标 / 推荐候选结论
- 关键规则
  - 一期以固定评分模板和人工汇总为主
  - 评分完成后生成不可直接覆盖的结果快照

### 8. 定标

- 输入
  - 评标结论
  - 定标审批意见
- 输出
  - 中标结果
  - 结果通知
- 关键规则
  - 定标结果必须有审批留痕
  - 定标前允许退回评标阶段补充说明

### 9. 归档

- 输入
  - 全流程附件
  - 操作日志
  - 结果文件
- 输出
  - 归档包
  - 可导出的过程材料
- 关键规则
  - 归档后项目进入只读主状态
  - 归档包应可按项目和标段追溯

## 异常流程

- 招标文件发布后发现错误
  - 通过澄清 / 更正版本处理，不直接覆盖历史版本
- 供应商报名不通过
  - 记录原因，可退回补充资料
- 截止时间前撤回投标
  - 保留撤回记录，不物理删除
- 开标时发现资料缺失
  - 记录异常并进入人工处理
- 评标结论需复核
  - 从定标前退回评标阶段，保留退回原因

## 一期业务边界

- 供应商门户只承担轻协同职责
- 评标专家管理一期不做复杂外部专家库
- 不建设独立电子交易大厅或强隔离评标环境
